Allocation

题目

有n个房子，每个价格为A[i]，而你有B刀，问可买几套。

思路

秒。用贪心的思想，每次买最便宜的，直到买不起为止。
算法分两步：首先排序，然后一次遍历，直到余额不足以支付当前的房子。

复杂度

时间复杂度O(nlog(n))，n为房子数量。
空间复杂度O(1)。

代码

t = int(input())
for i in range(1, t + 1):
    num, money = map(int, input().split(" "))
    res = 0
    prices = sorted([int(s) for s in input().split(" ")])
    for j in range(num):
        if money - prices[j] >= 0:
            money -= prices[j]
            res += 1
        else:
            break
    print("Case #{}: {}".format(i, res))

Plates

题目

有N堆盘子，每堆有k个。每个盘子有自己的价值。小P想拿走p个盘子，注意，每堆拿走的盘子一定是最上面的那些。问拿走的盘子的价值是多少。
翻译：给定数组values[n][k]和正数p，需要数组中选择p项。选择的规则是：如果选择values[i][j]，那么一定要选择values[i][0:j-1]。求最大和。

naive的心路历程

不得不吐槽一下，我第一反应是贪心法，用n个指针指向n个堆，每次选择值最大的，移动该堆的指针。而且我错误地做法遭遇了TLE，于是我一往无前地继续优化orz，改用heap来做，类似于合并k个链表。就是建立最大堆，每次pop出指最大的盘子。终于我WA了，清醒一点才发现这种贪心的思想并行不通。我在这道题折腾了很久，直到万念俱灰万籁俱寂，无奈之下暴力过了小数据量。

回看

啊，如此明显的dp！我感受到了我飞速的进步。首先求values的前缀和数组，然后二维dp。不过挺容易写错的，加一减一什么的。最后决定将suf数组和dp数组都补一圈0，这样两者之间依旧是对齐的，写起来方便一点。

dp推导

dp[i][j]表示在前i堆（从1开始）的盘子拿出j个（从1开始）的最大值。我们可以选择在第i堆拿走m∈[0, j]个盘子，拥有的价值为suf[i][j]。我们还需要从前i-1堆拿走p-j个盘子，也就是再加上dp[i-1][j-m]的价值。注意，m不能超过第i堆盘子的总个数，所以准确来说，m ∈ [0, min(j, k)]。

for i [1, N]:
 for j [1, P]:
  dp[i][j] := 0
   for m [0, min(j, K)]:
    dp[i][j] = max(dp[i][j], dp[i-1][j-m]+suf[i][m])

复杂度分析

时间复杂度：O(nkp)，三重循环。
空间复杂度：O(np)，dp数组的大小。

代码

t = int(input())
for c in range(1, t + 1):
    n, k, p = map(int, input().split(" "))
    values = []
    for j in range(n):
        values.append([int(s) for s in input().split(" ")])
    # suffix
    suf = [[0] * (k + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for j in range(1, k + 1):
            suf[i][j] = suf[i][j - 1] + values[i - 1][j - 1]
    # dp
    dp = [[0] * (p + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for j in range(1, p + 1):
            for m in range(min(j + 1, k + 1)):
                dp[i][j] = max(dp[i][j], dp[i - 1][j - m] + suf[i][m])
    print('Case #{}: {}'.format(c, dp[n][p]))

Workout

题目

小T准备上n节健身课，每节课为M[i]分钟，M是严格递增的。她可以在其中再插入k节课，时长自选。健身项目的难度定义为相邻课的时长差的最大值。问小T能设计出的最低难度是多少。

非正规解法

我当时又角度清奇。我是这么想的，举个栗子，比如sessions为[10 13 15 16 17]，能补两次。先算出差值矩阵init为[3, 2, 1, 1]，待除矩阵div = [1, 1, 1, 1]，两者pixelwise除一下，得到难度矩阵diff = [3, 2, 1, 1]。我们每次找diff中最大的，然后将它对应的div+1，算出新的diff。我们插入一次课，有div = [2, 1, 1, 1]，diff = [2, 2, 1, 1]。再插入一次（选择哪个2为最大值不重要），有div = [3, 1, 1, 1]，diff = [1, 2, 1, 1]，答案就是max(diff) = 2。
经检验，这个思路是正确的，不过时间复杂度是O(kn)，TLE警告。所以我选择用heap优化，降到了O(klog(n)) = O(1e5 log(1e5))。耶最终AC了。（赛前我做了heap的专题，所以真的很喜欢朝着heap靠orz

from math import ceil
import heapq

t = int(input())
for i in range(1, t+1):
    n, k = map(int, input().split(" "))
    sessions = [int(s) for s in input().split(" ")]
    diff = []
    for j in range(1, n):
        diff.append(sessions[j]-sessions[j-1])
    diff_div_init = [(-elem, 1, elem) for elem in diff]
    heapq.heapify(diff_div_init)
    while(k > 0):
        k -= 1
        currdiff, currdiv, init = heapq.heappop(diff_div_init)
        heapq.heappush(diff_div_init, (-ceil((init) / (currdiv + 1)), currdiv + 1, init))
    currdiff, currdiv, ind = heapq.heappop(diff_div_init)
    print('Case #{}: {}'.format(i, -currdiff))

二分

时光荏苒，岁月如梭。通过这俩月的学习，我一眼就能看出这题是很典型的“对值的二分”，好励志。我们的目标是寻找最小的最大差值d，who还能满足切分次数k' <= k。d初始化的区间为[1, max_diff]，在这个区间内二分即可。时间复杂度为O(nlog(max_diff)) = O(nlog(m)) = O(1e5 log(1e9))。

相似题目推荐：378. 有序矩阵中第K小的元素、410. 分割数组的最大值。刷过几道同类题后，手感就来了。

def valid(sess, mid, k):
    for i in range(1, len(sess)):
        diff = sess[i] - sess[i - 1]
        if diff > mid:
            # notice here, also can be written as
            # k -= ceil(diff / mid) - 1
            k -= (diff - 1) // mid
            if k < 0:
                return False
    return True

t = int(input())
for c in range(1, t+1):
    n, k = map(int, input().split(" "))
    sess = [int(s) for s in input().split(" ")]
    low, high = 1, 1
    for i in range(1, n):
        high = max(high, sess[i] - sess[i - 1])
    while low < high:
        mid = low + (high - low) // 2
        if valid(sess, mid, k):
            high = mid
        else:
            low = mid + 1
    print('Case #{}: {}'.format(c, low))

Bundling

题目

有n个字符串，对它们进行分组，每组k个字符串，保证可均分。每组的分数是组内所有字符串的最长公共前缀的长度，求总分数。

思路

当时也知道是前缀树，但不知道下一步该怎么求。现在想想，依旧没想到：)。参考了这里的解题思路，才想明白的。记每个前缀字符被共享的次数为$count$，那么这个字符的最大贡献为$$\lfloor \frac{cnt}{k} \rfloor$$，所以总分数为：
$$\Sigma_i \lfloor \frac{cnt_i}{k} \rfloor$$
另外，Trie的add函数如果使用递归，会超时，所以改用了迭代。

class TrieNode:
    def __init__(self):
        self.cnt = 0
        self.children = [None] * 26

    def add(self, s):
        self.cnt += 1
        if not s:
            return
        ch = s[0]
        i = ord(ch) - ord('A')
        if not self.children[i]:
            self.children[i] = TrieNode()
        self.children[i].add(s[1:])


t = int(input())
for c in range(1, t+1):
    n, k = map(int, input().split())
    ss = []
    for _ in range(n):
        ss.append(input())
    # Build Trie Tree
    root = TrieNode()
    base = ord('A')
    for s in ss:
        t = root
        # use iterative way instead of recursively
        for ch in s:
            t.cnt += 1
            i = ord(ch) - base
            if not t.children[i]:
                t.children[i] = TrieNode()
            t = t.children[i]
        t.cnt += 1
    # Traverse the tree
    res = 0
    stack = [root]
    root.cnt = 0
    while len(stack):
        curr = stack.pop()
        res += curr.cnt // k
        for child in curr.children:
            if child and child.cnt >= k:
                stack.append(child)
    print('Case #%d: %d' % (c, res))

#include <bits/stdc++.h>
#include <iostream>
using namespace std;

#define ll long long;
#define ar array;

int n, b, a[1000000];

void solve(){
    cin >> n >> b;
    for (int i = 0; i < n; i++){
        cin >> a[i];
    }
    sort(a, a + n);
    int ans = 0;
    for (int i = 0; i < n; i ++){
        if (b >= a[i]){
            b -= a[i];
            ++ans;
        }
    }
    cout << ans << "\n";
}

int main()
{
    ios::sync_with_stdio(0);
    cin.tie(0);
    int t, i = 1;
    cin >> t;
    while (t --){
        cout << "Case #" << i << ": ";
        solve();
        ++i;
    }
    return 0;
}
